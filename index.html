PG Dissertation Management System

it is a web-based application that helps postgraduate students, guides, and administrators manage the entire dissertation process in one place

Main Goals:

Allow students to submit topics, upload documents (PDFs), and track progress.h3

Let guides approve/reject topics, give feedback, and monitor milestones.

Provide admin with full control, analytics, and a centralized database of dissertations.

Ensure ethics clearance workflow and prevent duplicate topics.

Project Blueprint / Architecture

[Frontend UI]
↓
HTML, CSS, JavaScript, Bootstrap (Forms, dashboards, charts)
[Backend Logic]
↓
Java (Spring Boot OR JSP/Servlets) Handles requests, processes logic, validates data, sends queries to DB
[Database]
↓
MySQL
Stores:
- User details (students, guides, admin)
- Dissertation topics
- Ethics approval status
- Progress milestones
- File paths for uploaded PDFs

[File Storage]
↓
Local folder / Cloud (Google Drive)
Stores uploaded PDFs & returns links

[Extra Features]
↓
- Gantt chart for timeline tracking
- Automated email/notification alerts
- Duplicate topic detection (string similarity check)
